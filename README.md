# Automounter

AI-монтажёр видео: анализирует дроновые съёмки, понимает запрос на естественном языке и автоматически собирает музыкальный клип с синхронизацией по ритму.

---

## Требования

- **Node.js** 18+
- **Ollama** с установленной vision-моделью:
  ```
  ollama pull llava:13b
  ollama serve
  ```
- macOS с Apple Silicon рекомендуется (аппаратное кодирование видео). Linux/Windows использует `libx264`.

## Установка

```bash
npm install
```

## Использование

### Интерактивный режим (по умолчанию)

```bash
npm start
```

При первом запуске откроется мастер настройки. При последующих запусках предлагаются предыдущие значения по умолчанию (сохраняются в `config.json`):

```
  ╔════════════════════════════════════════╗
  ║        A U T O M O U N T E R          ║
  ║           AI Video Editor              ║
  ╚════════════════════════════════════════╝

  Input folder     [not set]
  > /Volumes/DroneDisk/DCIM/DJI_001

  Audio track      [not set]
  > ./track.mp3

  Editing prompt   [not set]
  > Эпичный ролик с природой, 23-27 августа 2025

  Duration (sec)   [60]
  >

  Bitrate (Mbps)   [auto]
  >
  ...
  Start? [Y/n]
```

### Пакетный / скриптовый режим

Передайте `--batch` (или пустой stdin) чтобы пропустить мастер настройки:

```bash
npm start -- --batch \
  --input  /Volumes/DroneDisk/DCIM/DJI_001 \
  --audio  ./track.mp3 \
  --prompt "Эпичный ролик с природой, 23-27 августа 2025" \
  --duration 60
```

### Индексация без монтажа

Предварительно проиндексировать папку (Vision AI анализ, результаты кэшируются):

```bash
# Индексировать новые/изменённые файлы (кэш существующих сохраняется)
npm start -- --index-only --input /Volumes/DroneDisk/DCIM/DJI_001

# Переиндексировать всё с нуля (удаляет .json сайдкары)
npm start -- --reindex --input /Volumes/DroneDisk/DCIM/DJI_001
```

## Логика эффектов

| Стиль аудио | Длина клипа   | Переходы                       | Slow-mo |
|-------------|---------------|--------------------------------|---------|
| `epic`      | 4–8 ударов    | flashIn на спадах энергии      | нет     |
| `dynamic`   | 4–8 ударов    | cut на высокодинамичных клипах | нет     |
| `lyrical`   | 8–16 ударов   | fade / dissolve                | да      |
| `calm`      | 8–16 ударов   | fade / dissolve                | да      |

Первый сегмент всегда получает `fadeIn`. Последний — всегда `fadeOut`.

Дополнительные проходы:
- **Color harmony**: соседние сегменты с разностью цвета >120 (RGB) → `flashIn`
- **Subject continuity**: смена группы пейзажа ≥2 шагов (напр. вода↔город) → `dissolve`

## Кэширование

Результаты Vision AI хранятся в `<videofile>.json` рядом с каждым исходным файлом. Кэш автоматически инвалидируется при:
- Изменении размера исходного файла
- Изменении версии схемы кэша (`_cacheVersion` в `indexer.ts`)

Для принудительной переиндексации: `npm start -- --reindex --input <папка>`

---

## Proxy-файлы (.lrf)

DJI и ряд других камер создают рядом с 4K-видео облегчённые proxy-копии с расширением `.lrf`. Automounter автоматически использует их для:
- Анализа чёрных кадров и смены сцен
- Извлечения ключевых кадров для Vision AI

Оригинальный файл используется только для рендеринга финальных сегментов.

## Структура файлов

```

src/audio.ts      MP3→WAV, определение ударов, анализ стиля/энергии
src/indexer.ts    Vision AI анализ видео, управление .json кэшем
src/director.ts   Multi-pass планирование монтажа, выбор эффектов
src/render.ts     FFmpeg рендеринг сегментов, сборка финального видео
src/types.ts      TypeScript интерфейсы и типы
src/ui.ts         Интерактивный визард и прогресс-бар
```
